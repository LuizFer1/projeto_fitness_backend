## Codebase Patterns
- PHP binary is at /tmp/php — use `export PATH="/tmp:$PATH"` before running php/artisan commands
- Route model binding for username is in routes/api.php via Route::bind('username', ...) — NOT via getRouteKeyName on User model
- User model uses UUID as primary key (HasUuids trait), foreign keys are `user_uuid`
- FriendshipRepository has statusBetween(), areFriends(), isBlocked() — use for all friendship checks
- PublicProfileResource uses fluent ->withFriendshipStatus() to inject friendship status
- UseCases live in app/Application/UseCases/{Domain}/
- Resources are namespaced by domain: app/Http/Resources/PublicProfile/, app/Http/Resources/Gamification/
- LevelCalculator::nameFor($level) returns Portuguese level names (Iniciante, Dedicado, etc.)
- All API error responses must use shape: { "error": { "code": "...", "message": "...", "details": {} } }
- Tests use RefreshDatabase trait; factories exist for User, Goal, Achievement, UserAchievement, Friendship, XpTransaction
- Friendship model needs HasFactory trait for FriendshipFactory to work
- throttle:api rate limiter is NOT defined — don't use it in route middleware
- For 404 with custom JSON shape, use response()->json([...], 404) not abort(404)
- Route parameter {username} resolves to User via Route::bind in api.php — controller receives User $username
- Goal model uses user_uuid FK; GoalFactory has public(), friends(), private(), completed(), archived() state methods
- Goals table has visibility (public/friends/private) and status (active/completed/archived/expired) columns
