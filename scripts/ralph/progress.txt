## Codebase Patterns
- Use `/tmp/php` or `export PATH="/tmp:$PATH"` to run PHP in this environment
- Tests use SQLite in-memory (phpunit.xml: DB_CONNECTION=sqlite, DB_DATABASE=:memory:) since MySQL is not available in CI
- phpunit.xml env vars need `force="true"` to override .env values (DB_CONNECTION, DB_DATABASE)
- Feature tests use `RefreshDatabase` trait and `User::factory()->create()` + `actingAs()` for auth
- User model uses HasUuids on `uuid` column — always use `$user->uuid` when creating related records
- Use PHPUnit 11 attributes (#[DataProvider(...)]) instead of doc-comment annotations to avoid deprecation warnings
- Main branch only has Init Project commit — gamification branch builds from scratch
- Foreign keys on users reference `uuid` column, not `id` — use `foreignUuid('user_uuid')->references('uuid')->on('users')`
- For timezone-dependent queries, do conversion in PHP/Carbon (SQLite doesn't support CONVERT_TZ)
- AddXpUseCase is the single entry point for awarding XP — always use it (or ProcessXpTransactionJob) rather than creating XpTransaction directly
- SQLite unique constraint errors say "UNIQUE constraint failed", MySQL uses SQLSTATE 23000 — handle both in catch blocks
