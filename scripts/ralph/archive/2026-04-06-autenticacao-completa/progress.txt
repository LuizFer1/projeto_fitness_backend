## Codebase Patterns
- PHP binary is at `/tmp/php` (not in PATH) — use `/tmp/php` for all artisan/pint commands
- Lint with `/tmp/php /workspace/vendor/bin/pint <files>` to auto-fix before committing
- AuthController is at `app/Http/Controllers/AuthController.php` — flat namespace, no Auth subfolder
- Sanctum tokens: `$user->tokens()` gives all tokens, `currentAccessToken()` gives the current one
- Routes use `auth:sanctum` middleware group in `routes/api.php`
- User model stores password as `password_hash` (not `password`) — use this column name everywhere
- User model has `Notifiable` trait — required for email-based features (password reset, email verification)
- Users table has NO `remember_token` column — avoid features that rely on it
- Password reset config is in config/auth.php `passwords.users` — token expires in 60 min
- Email verification uses `verified` middleware alias (Laravel built-in EnsureEmailIsVerified) — returns 403 JSON for API requests
- verify-email route is public (unauthenticated) — takes `id` (uuid) and `hash` (sha1 of email) params
- resend-verification requires `auth:sanctum` but NOT `verified` middleware
- Rate limiters are defined in `AppServiceProvider::boot()` using `RateLimiter::for()` — apply via `throttle:<name>` middleware
- Global API rate limit wraps all routes; specific limiters (login, register) stack on top as additional middleware
