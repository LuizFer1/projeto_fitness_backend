## Codebase Patterns
- PHP binary is at `/tmp/php` — use `export PATH="/tmp:$PATH"` before running php commands
- Laravel 12 uses `bootstrap/app.php` for middleware/exception config (no Kernel.php)
- API routes in `routes/api.php`, public routes go outside `auth:sanctum` middleware group
- Controllers in `app/Http/Controllers/Api/` for API-specific controllers
- Run `php vendor/bin/pint <files>` to auto-fix lint issues before committing
- The l5-swagger package has a broken config — `php artisan route:list` fails; use syntax checks instead
- Use `collect()->every()` for checking all items in a collection match a condition
- Register global middleware in `bootstrap/app.php` via `$middleware->append()` (Laravel 12)
- Use `Log::shareContext()` to add context (e.g., request_id) to all log entries in a request
- Exception rendering in bootstrap/app.php: type-hinted closures with `$exceptions->render()`, return null for non-API fallthrough
- API error format: `{ error: { code, message, fields } }` — all API exceptions follow this structure
- User password field is `password_hash` — use `Hash::check($input, $user->getAuthPassword())` for verification
- WorkoutExerciseLog links to WorkoutLog (not User) — delete via workout_log_id when cleaning user data
- Feature tests use SQLite in-memory DB — MySQL-specific SQL (ENUM MODIFY, CREATE OR REPLACE VIEW, CURDATE) must be guarded with `if (DB::connection()->getDriverName() !== 'sqlite')`
- DailyActivityLimit `date` column must NOT have a `'date'` cast — SQLite stores dates as `'Y-m-d H:i:s'` text causing `firstOrCreate` mismatches
- Achievements table is seeded in migration — use existing records in tests instead of creating duplicates
- When adding new ENUM values, also add them to the original migration's ENUM definition (SQLite can't ALTER CHECK constraints)
- API routes in tests need `/api/` prefix (e.g., `/api/register`, `/api/login`)
- AuthController `login()` method injects GamificationService — was missing import, fixed
- l5-swagger config references `\L5Swagger\Generator::OPEN_API_DEFAULT_SPEC_VERSION` which doesn't exist — replaced with literal `'3.0.0'`
- `RecalculateLeaderboardJob` was referenced in `routes/console.php` but didn't exist — created stub
