## Codebase Patterns
- Migrations use anonymous classes (`return new class extends Migration`)
- All tables use UUID primary keys with `$table->uuid('id')->primary()`
- Foreign keys use `$table->foreignUuid('col')->references('id')->on('table')->cascadeOnDelete()`
- Models use `HasUuids` trait for UUID generation
- No PHP runtime or Docker available in CI — cannot run migrations or artisan commands directly
- Project uses Laravel Pint for code style, no phpstan configured
- Soft deletes use `$table->softDeletes()` in migrations and `SoftDeletes` trait in models
- Models use `$fillable` for mass assignment; some lightweight models use `$guarded = []`
- Relationship pattern: `belongsTo(Model::class, 'foreign_key')` for inverse; `hasMany(Model::class)` for collections
- PostLike has `$timestamps = false` since it only has `created_at` (no `updated_at`)
- Controllers go in `app/Http/Controllers/Api/V1/` for v1 API endpoints
- Routes for v1 endpoints go inside the `Route::prefix('v1')` group in `routes/api.php`
- Use inline `$request->validate()` for validation (no FormRequest classes)
- Friendship queries must check both directions: (requester_id, addressee_id) OR (addressee_id, requester_id)
- Use `cursorPaginate()` for feed/timeline endpoints, `paginate()` for list endpoints
- PostController uses `formatPost()` helper to include like_count, comment_count, is_liked_by_me in all post responses
- Like toggle: check for existing PostLike, delete if exists, create if not — return `liked` boolean + updated count
- Comment deletion scopes by both `post_id` and `id` to ensure comment belongs to the post
- Gamification leaderboard controllers go in `app/Http/Controllers/Api/V1/Gamification/` namespace
- Leaderboard response format: `{ period, rankings: [{ position, user_id, name, avatar_url, period_xp, level, total_xp }], my_position }`
- User display name: prefer `nickname`, fallback to `name . ' ' . last_name`
