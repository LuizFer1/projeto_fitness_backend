## Codebase Patterns
- Controllers in Api/V1 use dependency injection, return JsonResponse, and follow namespace matching directory structure
- RankingController has reusable buildRanking() and getUserPosition() logic for leaderboard queries
- UserGamification model columns for XP: current_week_xp, current_month_xp, xp_total
- Cache TTL for rankings is 300 seconds (5 min) — use Cache::remember
- Models use HasUuids trait; user_id is string type
- No PHP runtime in dev container — quality checks are static only
- Route model binding for {username} resolves to User instance — controller param must be `User $username`
- User model field is `nickname` (not `username`) — the route binding queries `username` column which may need migration alignment
- Level thresholds are private constants in GamificationService — any controller needing them must duplicate the array
