## Codebase Patterns
- PHP binary is at `/tmp/php` (not in PATH)
- No PHPStan available; use `/tmp/php -l` for syntax check and `PATH="/tmp:$PATH" /tmp/php vendor/bin/pint --test` for linting
- Migrations use anonymous classes (`return new class extends Migration`)
- All tables use UUID primary keys with `HasUuids` trait on models
- Foreign UUIDs use `foreignUuid()->references('id')->on('table')->cascadeOnDelete()`
- AiPlan model uses `$guarded = []` and `$timestamps = false`
- Existing ai_plans table already has `version` column (smallInteger, default 1)
- `artisan` commands fail due to L5-Swagger config issue - create files manually
- Plan-related routes are under `Route::prefix('v1/plans/{planId}')` group in api.php
- Async plan generation pattern: create AiPlan with 'generating' status, dispatch job, return plan ID as job_id
- MySQL ENUM column changes require raw SQL `ALTER TABLE ... MODIFY COLUMN` in migrations
